# TCLProductAgent - 智能产品咨询助手技术方案

## 一、项目概述

### 1.1 项目名称
**TCLProductAgent** - TCL智能产品咨询助手

### 1.2 项目背景
TCL作为全球领先的消费电子品牌，拥有电视、空调、冰箱、洗衣机等丰富的产品线。消费者在选购产品时面临以下痛点：
- 产品型号众多，参数复杂，难以理解
- 不清楚哪款产品最适合自己需求
- 产品说明书晦涩难懂，查询不便
- 官网客服响应慢，门店咨询成本高

本项目旨在构建一个基于AI Agent的智能产品咨询系统，帮助消费者快速了解TCL产品、获取个性化推荐、解答使用疑问。

### 1.3 核心价值
- **提升用户体验**：24/7即时响应，智能问答
- **降低服务成本**：自动化客服，减少人工成本
- **促进销售转化**：精准产品推荐，缩短决策周期
- **展示AI能力**：体现TCL智能化转型成果

### 1.4 应用场景
1. **官网在线客服**：嵌入TCL官网，替代传统客服
2. **门店智能助手**：线下门店配备智能咨询终端
3. **微信小程序**：移动端产品咨询服务
4. **内部培训工具**：帮助销售人员快速掌握产品知识

## 三、核心功能模块

### 3.1 产品知识问答（核心功能）

**功能描述**：
用户输入产品相关问题，Agent自动检索产品知识库并生成回答。

**技术实现**：
```
用户问题 → 意图识别 → 知识检索(RAG/关键词) → 答案生成 → 来源引用 → 输出
```

**流程详解**：
1. **问题预处理**：提取关键词，识别产品类别
2. **向量检索**：在ChromaDB中检索相关知识片段（优先）
3. **关键词检索**：RAG超时时使用关键词匹配作为fallback
4. **上下文构建**：拼接检索结果 + 用户问题
5. **答案生成**：GLM-4生成自然语言回答
6. **来源引用**：标注回答来源，低相关度来源自动过滤

**示例对话**：
```
用户：TCL电视保修政策是什么？
Agent：根据提供的文档信息，TCL电视的保修政策如下：
      - 大多数TCL电视提供整机1年保修
      - 部分高端型号还提供主要部件3年保修

【来源引用】
[1] TCL产品手册 - 保修条款
```

### 3.2 智能产品推荐

**功能描述**：
根据用户需求（预算、用途、空间等），推荐最合适的TCL产品。

**技术实现**：
```
用户需求 → 需求解析 → 产品筛选 → 相关性评分 → 推荐排序 → 输出
```

**推荐算法**：
- **产品类型识别**：智能识别电视/空调/冰箱/洗衣机
- **价格范围提取**：从自然语言中提取预算区间
- **相关性评分**：综合匹配度评分（价格30% + 功能40% + 场景30%）
- **智能筛选**：只推荐相关度高的产品

**示例对话**：

### 5.1 RAG检索增强生成

**原理**：
结合检索（Retrieval）和生成（Generation），提升回答准确性。

**实现步骤**：
```python

)

# 2. 向量化存储（带缓存）
```

### 5.2 多层级缓存机制

**缓存架构**：
```python
class LRUCacheWithTTL:
    def __init__(self, max_size: int = 100, ttl_seconds: int = 300):
        self.cache = OrderedDict()
        self.max_size = max_size
        self.ttl_seconds = ttl_seconds
    
    def get(self, key: str) -> Optional[Any]:
        # 检查过期时间，自动清理
        if time.time() > item['expire_time']:
            del self.cache[key]
            return None
        self.cache.move_to_end(key)
        return item['value']
    
    def set(self, key: str, value: Any):
        # LRU淘汰策略
        if len(self.cache) >= self.max_size:
            self.cache.popitem(last=False)
        self.cache[key] = {
            'value': value,
            'expire_time': time.time() + self.ttl_seconds
        }
```

**缓存层级**：

| 缓存类型       | 容量  | TTL   | 说明                      |
| -------------- | ----- | ----- | ------------------------- |
| LLM回答缓存    | 100条 | 5分钟 | 支持相似度匹配（阈值0.6） |
| RAG检索缓存    | 100条 | 5分钟 | 向量检索结果              |
| 关键词搜索缓存 | 50条  | 5分钟 | 关键词匹配结果            |
| 文档加载缓存   | -     | -     | 文件哈希校验              |

**相似度匹配算法**：
```python
def compute_similarity(query1: str, query2: str) -> float:
    # 文本相似度（40%权重）
    text_sim = difflib.SequenceMatcher(None, query1, query2).ratio()
    
    # 关键词相似度（60%权重）
    keywords1 = extract_keywords(query1)
    keywords2 = extract_keywords(query2)
    keyword_sim = len(keywords1 & keywords2) / len(keywords1 | keywords2)
    
    return text_sim * 0.4 + keyword_sim * 0.6
```

### 5.3 意图识别与路由

**意图分类实现**：
```python
class Intent(Enum):
    PRODUCT_QUERY = "product_query"      # 产品查询
    RECOMMENDATION = "recommendation"    # 产品推荐
    COMPARISON = "comparison"            # 产品对比
    TROUBLESHOOTING = "troubleshooting"  # 故障排查
    TIPS = "tips"                        # 使用技巧
    GREETING = "greeting"                # 问候寒暄
    GENERAL_CHAT = "general_chat"        # 日常闲聊

class IntentClassifier:
    def classify(self, user_input: str) -> Intent:
        # 关键词匹配 + 规则优先级
        for intent, keywords in self.keyword_map.items():
            if any(kw in user_input for kw in keywords):
                return intent
        return Intent.GENERAL_CHAT
```

### 5.4 来源引用与相关性过滤

**相关性过滤**：
```python
def _format_sources_section(self, sources: List[Dict]) -> str:
    # 过滤低相关度来源（低于10%不显示）
    filtered = [s for s in sources if s.get('relevance', 0) >= 10]
    
    # 去重并重新编号
    seen = set()
    unique_sources = []
    for source in filtered:
        key = (source.get('type'), source.get('name'))
        if key not in seen:
            seen.add(key)
            source['id'] = len(unique_sources) + 1
            unique_sources.append(source)
    return format_sources(unique_sources)
```

| MAX_HISTORY_LENGTH | 20          | 最大对话历史长度                      |

### 8.3 模型选择建议

## 十、技术风险与应对

### 10.1 风险识别

| 风险          | 概率 | 影响 | 应对方案                              |
| ------------- | ---- | ---- | ------------------------------------- |
| GLM API响应慢 | 中   | 中   | 使用GLM-4.5-Air快速模型 + LLM回答缓存 |
| RAG检索效果差 | 中   | 高   | 优化切片策略，关键词检索作为fallback  |
| 数据质量问题  | 低   | 中   | 使用官网真实参数，人工审核            |
| 响应时间过长  | 中   | 中   | 多层级缓存机制，对话历史长度限制      |
| 内存占用过大  | 低   | 中   | LRU缓存淘汰策略，定期清理过期数据     |

## 十、技术亮点

1. **RAG检索增强生成**：结合知识库检索和LLM生成，提升回答准确性
2. **多层级缓存机制**：LRU+TTL缓存策略，支持相似度匹配，显著提升响应速度
3. **智能意图识别**：7种意图分类，自动路由到相应功能模块
4. **来源引用显示**：自动标注回答来源，低相关度来源过滤，提升可信度
5. **优雅降级**：RAG初始化超时时自动切换到关键词检索
6. **对话历史管理**：限制历史长度，防止token膨胀和内存泄漏
7. **多格式文档支持**：PDF、Markdown、JSON文档统一加载
