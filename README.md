# TCL智能产品咨询助手 (TCL Product Agent)

基于 LangChain 和 GLM-4 的智能客服 Agent 系统

***

## 📋 项目简介

TCL智能产品咨询助手是一个基于大语言模型的智能客服系统，旨在帮助消费者快速了解TCL产品、获取个性化推荐、解答使用疑问。

### 核心价值
- **提升用户体验**：24/7即时响应，智能问答
- **降低服务成本**：自动化客服，减少人工成本
- **促进销售转化**：精准产品推荐，缩短决策周期

### 应用场景
1. **官网在线客服**：嵌入TCL官网，替代传统客服
2. **门店智能助手**：线下门店配备智能咨询终端
3. **微信小程序**：移动端产品咨询服务
4. **内部培训工具**：帮助销售人员快速掌握产品知识

***

## 🎯 核心功能

### 1. 产品知识问答（核心功能）

基于RAG（检索增强生成）技术，提供准确的产品信息查询服务。支持向量检索和关键词检索双重模式，RAG超时时自动降级为关键词检索。

### 2. 智能产品推荐

根据用户需求（预算、用途、空间等）推荐最合适的产品。支持产品类型智能识别、价格范围提取、多维度相关性评分（价格30% + 功能40% + 场景30%）。

### 3. 产品参数对比

多维度对比产品参数，生成对比表格和购买建议。对比维度包括：基础参数、技术参数、功能特性、适用场景。

### 4. 故障排查指导

提供产品故障的自助排查步骤和建议，包含安全免责声明。不提供专业维修建议，建议联系官方售后。

### 5. 使用技巧建议

根据产品类型，提供使用技巧和保养建议。

### 6. 智能意图识别

自动识别用户意图（产品查询、推荐、对比、故障排查、使用技巧、问候寒暄、日常闲聊），并路由到相应功能模块。问候类问题直接响应，不调用LLM。

### 7. 多层级缓存优化

| 缓存类型       | 容量  | TTL   | 说明                      |
| -------------- | ----- | ----- | ------------------------- |
| LLM回答缓存    | 100条 | 5分钟 | 支持相似度匹配（阈值0.6） |
| RAG检索缓存    | 100条 | 5分钟 | 向量检索结果              |
| 关键词搜索缓存 | 50条  | 5分钟 | 关键词匹配结果            |
| 文档加载缓存   | -     | -     | 文件哈希校验              |

## 🚀 快速开始

### 1. 环境准备

```bash
# 创建Python环境
conda create -n tcl_agent python=3.12
conda activate tcl_agent

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置API密钥

```bash
# 复制环境变量模板（已包含默认API密钥，可直接使用）
cp .env.example .env

# 如果需要使用自己的API密钥，编辑.env文件
# ZHIPUAI_API_KEY=your_api_key_here
```

### 3. 运行应用

```bash
# 方式1：一键启动（推荐）
./setup.sh

# 方式2：手动启动
python src/app.py
```

访问地址: http://localhost:7860

## 📖 使用指南

### 模型选择

| 模型名称      | 特点                 | 推荐度 |
| ------------- | -------------------- | ------ |
| glm-4.5-air   | 响应速度快，稳定性好 | ⭐⭐⭐⭐⭐  |
| glm-4.7-flash | 最新模型，性能更强   | ⭐⭐⭐⭐   |

建议使用 `glm-4.5-air` 以获得最佳体验。

### 测试场景

| 场景     | 测试问题                 | 预期效果                     |
| -------- | ------------------------ | ---------------------------- |
| 产品查询 | TCL电视保修政策是什么？  | 返回保修政策说明 + 来源引用  |
| 产品推荐 | 推荐一款5000元左右的电视 | 返回推荐列表 + 产品对比      |
| 产品对比 | Q10L和T7H哪个好？        | 返回对比表格 + 购买建议      |

| 缓存效果 | 重复查询相同问题         | 第二次查询响应更快（毫秒级） |

### 配置说明

`.env` 文件可配置的参数：

| 参数               | 默认值      | 说明                  |
| ------------------ | ----------- | --------------------- |
| ZHIPUAI_API_KEY    | -           | 必须填写，GLM API密钥 |
| MODEL_NAME         | glm-4.5-air | 模型名称              |
| MAX_TOKENS         | 2000        | 最大输出token数       |
| TEMPERATURE        | 0.7         | 生成温度              |
| TOP_K_RETRIEVAL    | 3           | 检索返回条数          |
| CACHE_MAX_SIZE     | 100         | 缓存最大条数          |
| CACHE_TTL_SECONDS  | 300         | 缓存过期时间（秒）    |
| MAX_HISTORY_LENGTH | 20          | 最大对话历史长度      |

## 🎯 技术亮点

1. **RAG检索增强生成**: 结合知识库检索和LLM生成，提升回答准确性
2. **多层级缓存机制**: LRU+TTL缓存策略，支持相似度匹配，显著提升响应速度
3. **智能意图识别**: 7种意图分类，自动路由到相应功能模块
4. **来源引用显示**: 自动标注回答来源，低相关度来源过滤，提升可信度
5. **优雅降级**: RAG初始化超时时自动切换到关键词检索
6. **对话历史管理**: 限制历史长度，防止token膨胀和内存泄漏
7. **多格式文档支持**: PDF、Markdown、JSON文档统一加载
