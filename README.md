# TCL智能产品咨询助手 (TCL Product Agent)

基于 LangChain 和 GLM-4 的智能客服 Agent 系统

***

## 📋 项目简介

TCL智能产品咨询助手是一个基于大语言模型的智能客服系统，能够帮助用户：

- 📺 查询TCL产品信息（电视、空调、冰箱、洗衣机）
- 🎯 获取智能产品推荐
- 📊 对比不同产品型号
- 🔧 排查产品故障
- 💡 获取使用技巧和保养建议
- 👋 支持问候寒暄等日常对话

***

## 🎯 核心功能

### 1. 产品知识问答

基于RAG（检索增强生成）技术，提供准确的产品信息查询服务

### 2. 智能产品推荐

根据用户需求（预算、用途、空间等）推荐最合适的产品，支持相关性评分和多维度筛选

### 3. 产品参数对比

多维度对比产品参数，生成对比表格和购买建议

### 4. 故障排查指导

提供产品故障的自助排查步骤和建议，包含安全免责声明

### 5. 使用技巧建议

提供产品使用技巧和保养建议

### 6. 智能意图识别

自动识别用户意图（产品查询、推荐、对比、故障排查、使用技巧、问候寒暄、日常闲聊），并路由到相应功能模块

### 7. 多层级缓存优化

- RAG检索结果缓存（LRU+TTL）
- LLM回答缓存（支持相似度匹配）
- 文档加载文件哈希校验
- 关键词搜索结果缓存

***

## 🏗️ 技术架构

```
用户交互层 (Gradio) → Agent核心层 (LangChain) → 知识存储层 (ChromaDB)
                      ↓                    ↓
                 GLM-4.5-Air (智谱AI)   缓存层 (LRU+TTL)
```

### 技术栈

| 组件 | 技术 |
|------|------|
| 大语言模型 | GLM-4.5-Air / GLM-4.7-Flash (智谱AI) |
| Agent框架 | LangChain 1.4+ |
| 向量数据库 | ChromaDB |
| 文档解析 | pdfplumber, markdown, json |
| Web界面 | Gradio 4.0+ |
| 编程语言 | Python 3.12+ |

### 缓存架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户查询                              │
└──────────────┬─────────────────────────────────────────┘
               │
   ┌───────────┼───────────┐
   ▼           ▼           ▼
LLM回答缓存 RAG检索缓存 关键词搜索缓存
(相似度匹配) (LRU+TTL)  (LRU+TTL)
   │           │           │
   └───────────┼───────────┘
               │
         文档加载缓存
      (文件哈希校验)
               │
               ▼
         磁盘文件读取
```

## 🚀 快速开始

### 1. 环境准备

```bash
# 创建Python环境
conda create -n tcl_agent python=3.12
conda activate tcl_agent

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置API密钥

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑.env文件，填入你的GLM API密钥
# ZHIPUAI_API_KEY=your_api_key_here
```

### 3. 运行应用

```bash
# 方式1：一键启动（推荐）
./setup.sh

# 方式2：手动启动
python src/app.py
```

访问地址: http://localhost:7860

## 📖 使用指南

### 模型选择

| 模型名称 | 特点 | 推荐度 |
|---------|------|--------|
| glm-4.5-air | 响应速度快，稳定性好 | ⭐⭐⭐⭐⭐ |
| glm-4.7-flash | 最新模型，性能更强 | ⭐⭐⭐⭐ |

建议使用 `glm-4.5-air` 以获得最佳体验。

### 测试场景

| 场景 | 测试问题 | 预期效果 |
|------|---------|---------|
| 产品查询 | TCL电视保修政策是什么？ | 返回保修政策说明 + 来源引用 |
| 产品推荐 | 推荐一款5000元左右的电视 | 返回推荐列表 + 产品对比 |
| 产品对比 | Q10L和T7H哪个好？ | 返回对比表格 + 购买建议 |
| 故障排查 | 电视开机没反应怎么办？ | 返回排查步骤 + 免责声明 |
| 问候寒暄 | 你好 | 直接返回友好问候，不调用LLM |
| 缓存效果 | 重复查询相同问题 | 第二次查询响应更快（毫秒级） |

### 配置说明

`.env` 文件可配置的参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| ZHIPUAI_API_KEY | - | 必须填写，GLM API密钥 |
| MODEL_NAME | glm-4.5-air | 模型名称 |
| MAX_TOKENS | 2000 | 最大输出token数 |
| TEMPERATURE | 0.7 | 生成温度 |
| TOP_K_RETRIEVAL | 3 | 检索返回条数 |
| CACHE_MAX_SIZE | 100 | 缓存最大条数 |
| CACHE_TTL_SECONDS | 300 | 缓存过期时间（秒） |
| MAX_HISTORY_LENGTH | 20 | 最大对话历史长度 |

## 🎯 项目亮点

1. **RAG检索增强生成**: 结合知识库检索和LLM生成，提升回答准确性
2. **多层级缓存机制**: LRU+TTL缓存策略，支持相似度匹配，显著提升响应速度
3. **智能意图识别**: 7种意图分类（产品查询、推荐、对比、故障排查、使用技巧、问候、闲聊）
4. **来源引用显示**: 自动标注回答来源，提升可信度
5. **相关性过滤**: 低相关度来源自动过滤，避免误导用户
6. **对话历史管理**: 限制历史长度，防止token膨胀
7. **优雅降级**: RAG初始化超时时自动切换到关键词检索
8. **多格式文档支持**: PDF、Markdown、JSON文档统一加载
